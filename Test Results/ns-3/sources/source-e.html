64 /**
65 This class extends NumberAxis and handles cycling.
66 
67 Traditional representation of data in the range x0..x1
68 <pre>
69 |-------------------------|
70 x0                       x1
71 </pre>
72 
73 Here, the range bounds are at the axis extremities.
74 With cyclic axis, however, the time is split in
75 "cycles", or "time frames", or the same duration : the period.
76 
77 A cycle axis cannot by definition handle a larger interval
78 than the period : <pre>x1 - x0 &gt;= period</pre>. Thus, at most a full
79 period can be represented with such an axis.
80 
81 The cycle bound is the number between x0 and x1 which marks
82 the beginning of new time frame:
83 <pre>
84 |---------------------|----------------------------|
85 x0                   cb                           x1
86 &lt;---previous cycle---&gt;&lt;-------current cycle--------&gt;
87 </pre>
88 
89 It is actually a multiple of the period, plus optionally
90 a start offset: <pre>cb = n * period + offset</pre>
91 
92 Thus, by definition, two consecutive cycle bounds
93 period apart, which is precisely why it is called a
94 period.
95 
96 The visual representation of a cyclic axis is like that:
97 <pre>
98 |----------------------------|---------------------|
99 cb                         x1|x0                  cb
100 &lt;-------current cycle--------&gt;&lt;---previous cycle---&gt;
101 </pre>
102 
103 The cycle bound is at the axis ends, then current
104 cycle is shown, then the last cycle. When using
105 dynamic data, the visual effect is the current cycle
106 erases the last cycle as x grows. Then, the next cycle
107 bound is reached, and the process starts over, erasing
108 the previous cycle.
109 
110 A Cyclic item renderer is provided to do exactly this.
111 
112  */

722 
723         double vmax = range.getUpperBound();
724         double vp = getCycleBound();
725 
726         double jmin = 0.0;
727         double jmax = 0.0;
728         if (RectangleEdge.isTopOrBottom(edge)) {

730             jmax = dataArea.getMaxX();
731         }

734             jmax = dataArea.getMinY();

738             double jbreak = jmax - (vmax - vp) * (jmax - jmin) / this.period;

743                 return vp - (java2DValue - jmin) * this.period / (jmax - jmin);

807                 return jmin + (value - vp) * (jmax - jmin) / this.period;
